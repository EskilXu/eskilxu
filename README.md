## Hi, I'm Eskil 👋

**Defensive infrastructure operator → agentic AI builder.** Tokyo-based.

I spent the last decade building things that protect: industrial-symbiosis networks at TEDA, 5G edge cloud at EdgeOn, and Web3 user safety at GoPlus Security (where I co-founded and operated international markets through February 2026, scaling the platform to ~30M daily API calls across 40+ chains). Now I'm spending 6–12 hours a day inside Claude Code and Claude CLI, working on the question that has followed me through every substrate I've operated in:

> **What does defensive infrastructure look like when the things being defended — and the things doing the attacking — are agents?**

I work in [Vitalik Buterin's d/acc framework](https://vitalik.eth.limo/general/2023/11/27/techno_optimism.html) — differentially accelerate the technologies that are defensive, decentralized, and accessible. The two pinned repositories below are early-stage, exploratory, and meant to be used as thinking tools more than as products. They will get more polished over time.
